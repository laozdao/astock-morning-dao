感谢您对 **Astock Morning DAO** 项目的关注！我们欢迎各种形式的贡献，包括但不限于：

## 🚀 如何贡献

### 1. 提交 Issue

如果您发现了 Bug 或有新功能建议，请通过 [GitHub Issues](https://github.com/laozdao/astock-morning-dao/issues) 提交。

**Bug 报告模板**：

```markdown
**问题描述**
清晰简洁地描述问题。

**复现步骤**
1. 步骤一
2. 步骤二
3. 步骤三

**期望行为**
描述您期望发生的行为。

**实际行为**
描述实际发生的行为。

**环境信息**
- 操作系统：
- 技能版本：
- 相关配置：

**附加信息**
截图、日志或其他相关信息。
```

**功能建议模板**：

```markdown
**功能描述**
清晰简洁地描述您建议的功能。

**使用场景**
描述这个功能会在什么场景下使用。

**预期效果**
描述这个功能带来的好处。

**实现思路**（可选）
如果您有实现思路，请简要说明。
```

### 2. 提交 Pull Request

#### 准备工作

1. **Fork 仓库**
   ```bash
   git clone https://github.com/YOUR_USERNAME/astock-morning-dao.git
   cd astock-morning-dao
   ```

2. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   # 或
   git checkout -b fix/your-bug-fix
   ```

3. **进行修改**
   - 确保代码风格一致
   - 添加必要的注释
   - 更新相关文档

4. **提交更改**
   ```bash
   git add .
   git commit -m "feat: 添加新功能描述"
   git push origin feature/your-feature-name
   ```

5. **创建 Pull Request**
   - 前往原仓库创建 PR
   - 填写详细的 PR 描述
   - 关联相关 Issue（如有）

#### 提交规范

**Commit Message 格式**：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式调整（不影响功能） |
| `refactor` | 代码重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建过程或辅助工具变动 |

**示例**：

```
feat(dimension): 添加新能源维度分析

- 新增新能源政策跟踪
- 添加产业链映射表
- 更新评分权重

Closes #123
```

## 📋 开发规范

### 代码风格

- 使用清晰的变量命名
- 添加必要的注释说明
- 保持代码简洁可读
- 遵循 SKILL.md 的格式规范

### 文档规范

- 更新 README.md 中的相关说明
- 如需，更新 docs/ 目录下的研究文档
- 确保文档中的链接有效

### 测试要求

- 提交前进行本地测试
- 确保技能可以正常打包
- 验证模板变量替换正确

## ❓ 常见问题

**Q: 我没有编程经验，可以贡献吗？**

A: 当然可以！您可以：
- 提交 Bug 报告
- 改进文档
- 分享使用经验
- 提出功能建议

**Q: 我的 PR 为什么没有被合并？**

A: 可能的原因：
- 与项目目标不符
- 代码质量不符合要求
- 缺少必要的文档
- 存在冲突需要解决

我们会尽快审核 PR，如有问题会给出反馈。

**Q: 我可以将这个技能用于商业项目吗？**

A: 可以！MIT 协议允许商业使用。但请注意：
- 保留版权声明
- 本技能仅供学习研究，不构成投资建议
