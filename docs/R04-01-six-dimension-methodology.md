# 六维度研究方法论：从信息收集到投资决策的系统化流程

> **一句话摘要**：构建从信息收集、数据处理、信号提取到投资决策的六维度研究方法论，实现投资研究的系统化、流程化和可复现。

---

## 一、引言

### 1.1 研究痛点

投资研究面临的核心挑战：

- **信息碎片化**：海量资讯缺乏结构化整理
- **分析主观化**：依赖个人经验，难以复现
- **决策随机化**：缺乏统一标准，受情绪影响大
- **复盘困难**：无法追溯决策依据，难以迭代优化

### 1.2 方法论目标

建立一套标准化的研究流程：

1. **结构化**：将非结构化信息转化为结构化数据
2. **量化化**：将定性判断转化为定量评分
3. **系统化**：建立从输入到输出的完整流程
4. **可复现**：任何研究者按流程可得到一致结论

---

## 二、六维度研究流程

### 2.1 整体流程图

```
信息收集 → 数据清洗 → 特征提取 → 信号生成 → 交叉验证 → 决策输出
   │          │          │          │          │          │
   ▼          ▼          ▼          ▼          ▼          ▼
宏观数据    异常处理    周期定位    评分(0-100)  共振识别   仓位建议
政策资讯    标准化      板块映射    方向判断    矛盾处理   板块配置
外围行情    对齐        映射关系    强度评级    置信评估   操作计划
资金流向    验证        趋势确认    时效判断              风险控制
情绪指标    补全        极端识别    预警信号              止损设置
技术形态    平滑        关键点位    突破/破位             动态调整
```

### 2.2 各阶段详解

#### 阶段一：信息收集（07:00-08:30）

**信息来源矩阵**：

| 维度 | 主要来源 | 辅助来源 | 更新频率 |
|------|----------|----------|:--------:|
| 宏观 | 国家统计局、央行 | Wind、同花顺 | 月度/季度 |
| 政策 | 证监会、国务院 | 财联社、证券时报 | 实时 |
| 外围 | 彭博、路透社 | 雅虎财经 | 实时 |
| 资金 | 港交所、交易所 | 东方财富 | 日度 |
| 情绪 | 交易所数据 | 韭研公社、雪球 | 日度 |
| 技术 | 交易软件 | TradingView | 实时 |

**信息质量评估**：

| 评估维度 | 权重 | 评估标准 |
|----------|:----:|----------|
| 权威性 | 30% | 官方>权威媒体>自媒体 |
| 时效性 | 25% | 实时>日内>隔夜 |
| 准确性 | 25% | 多源交叉验证 |
| 相关性 | 20% | 与投资决策的直接关联度 |

#### 阶段二：数据清洗（08:30-08:45）

**清洗流程**：

1. **异常值检测**：识别离群数据（如北向资金单日异常波动）
2. **缺失值处理**：使用前向填充或插值法
3. **标准化处理**：统一单位（如百分比、亿元）
4. **时间对齐**：确保各维度数据时间戳一致

**数据质量检查清单**：

- [ ] 数值范围合理（如涨跌幅在±20%内）
- [ ] 时间戳正确（非交易日数据标记）
- [ ] 单位统一（金额统一为亿元）
- [ ] 逻辑自洽（如资金流入与涨跌方向一致）

#### 阶段三：特征提取（08:45-09:00）

**特征工程**：

| 维度 | 原始数据 | 提取特征 | 特征含义 |
|------|----------|----------|----------|
| 宏观 | GDP、PMI | 周期阶段 | 复苏/过热/滞胀/衰退 |
| 政策 | 政策文本 | 受益板块 | 行业映射关系 |
| 外围 | 指数涨跌 | 开盘指引 | 对A股情绪影响 |
| 资金 | 净流入额 | 趋势强度 | 连续流入天数 |
| 情绪 | 涨跌家数 | 极端值 | 贪婪/恐惧指数 |
| 技术 | K线数据 | 关键点位 | 支撑/压力位 |

#### 阶段四：信号生成（09:00-09:15）

**信号定义**：

每个维度输出标准化信号：
- **方向**：看多/看空/中性
- **强度**：⭐1-5级
- **时效**：长期/中期/短期/日内
- **置信度**：高/中/低

**信号生成规则示例（资金维度）**：

| 北向流入 | 主力流向 | 信号方向 | 强度 |
|:--------:|:--------:|:--------:|:----:|
| >100亿 | 多板块流入 | 看多 | ⭐⭐⭐⭐⭐ |
| 50-100亿 | 热点流入 | 看多 | ⭐⭐⭐⭐ |
| 20-50亿 | 结构流入 | 看多 | ⭐⭐⭐ |
| <20亿 | 分散 | 中性 | ⭐⭐ |
| <-50亿 | 流出 | 看空 | ⭐⭐⭐ |

#### 阶段五：交叉验证（09:15-09:25）

**验证逻辑**：

| 验证类型 | 条件 | 处理方式 |
|----------|------|----------|
| 强共振 | ≥4维同向，含2个高优先级 | 高置信度，积极执行 |
| 弱共振 | 3维同向，含1个高优先级 | 中等置信度，谨慎执行 |
| 分歧 | 多空各半 | 低置信度，观望或降低仓位 |
| 矛盾 | 高优先级维度矛盾 | 以高优先级为准，标记风险 |

**矛盾处理决策树**：

```
维度矛盾？
  ├─ 是 → 高优先级维度矛盾？
  │       ├─ 是 → 以宏观/政策为准，降低仓位
  │       └─ 否 → 以高优先级为准
  └─ 否 → 正常执行
```

#### 阶段六：决策输出（09:25-09:30）

**决策要素**：

| 要素 | 内容 | 格式 |
|------|------|------|
| 市场判断 | 方向+强度 | 偏多（⭐⭐⭐⭐） |
| 仓位建议 | 具体比例 | 60-80% |
| 板块配置 | 重点板块+占比 | AI算力 30% |
| 操作计划 | 时段+动作 | 09:35低吸科技龙头 |
| 风险控制 | 止损位+减仓条件 | 跌破3100减仓50% |

## 五、常见错误与规避

### 5.1 数据层面

| 错误类型 | 典型案例 | 规避方法 |
|----------|----------|----------|
| 幸存者偏差 | 仅分析涨停股 | 纳入全部样本 |
| 前视偏差 | 使用未来数据 | 严格时间对齐 |
| 异常值未处理 | 北向资金单日千亿 | 设置合理阈值 |
| 数据频率不一致 | 日线与分钟线混用 | 统一频率或降采样 |

### 5.2 分析层面

| 错误类型 | 典型案例 | 规避方法 |
|----------|----------|----------|
| 过度拟合 | 参数过多 | 交叉验证、正则化 |
| 维度遗漏 | 忽视政策维度 | 强制检查清单 |
| 权重主观 | 凭感觉设权重 | 回测优化或等权 |
| 忽视时效 | 使用过期数据 | 时间戳检查 |

### 5.3 决策层面

| 错误类型 | 典型案例 | 规避方法 |
|----------|----------|----------|
| 仓位过重 | 单票满仓 | 分散化、仓位上限 |
| 追涨杀跌 | 情绪驱动交易 | 严格执行计划 |
| 止损犹豫 | 破位不止损 | 预设止损位 |
| 频繁交易 | 日内多次调仓 | 设定调仓频率 |

## 六、进阶技巧

### 6.1 动态权重调整

**基于市场状态的权重调整**：

| 市场状态 | 调整策略 | 调整后的权重 |
|----------|----------|--------------|
| 政策密集期 | 提高政策权重 | 政策35%，宏观20% |
| 财报季 | 提高宏观权重 | 宏观35%，政策20% |
| 震荡市 | 提高技术权重 | 技术15%，情绪15% |
| 趋势市 | 提高资金权重 | 资金25%，技术5% |

### 6.2 信号融合算法

**加权平均法**：

$$\text{综合信号} = \sum_{i=1}^{6} w_i \times s_i$$

其中 $w_i$ 为维度权重，$s_i$ 为维度信号（看多=1，中性=0，看空=-1）。

**贝叶斯融合法**：

$$P(H|E) = \frac{P(E|H) \times P(H)}{P(E)}$$

将各维度视为证据，更新对市场方向的后验概率。

```python
def generate_signal(value, thresholds):
    """
    根据阈值生成信号
    thresholds: [看空阈值, 中性阈值, 看多阈值]
    """
    if value < thresholds[0]:
        return -1  # 看空
    elif value < thresholds[1]:
        return 0   # 中性
    else:
        return 1   # 看多
```
